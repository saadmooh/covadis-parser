# كيف يخزن Covadis بيانات شبكة الصرف الصحي في ملفات DXF

## نظرة عامة

ملف DXF الخاص بـ Covadis يحتوي على بيانات شبكة الصرف الصحي (EU = Eaux Usées = مياه مستعملة) موزعة عبر عدة **layers** (طبقات). 

**ملفان محللان:**
- `network 1.dxf` — شبكة EU 1 الأساسية (3 فروع، 137 منهول، بدون تسميات)
- `networks 1+2.dxf` — الشبكة الكاملة (EU 1 + تسميات + DN 200 + assai)

## 1. Layers خاصة بشبكة EU (الصرف الصحي)

### 1.1 `EU 1_Canalisations` — الخطوط الهندسية للأنابيب

- **النوع**: LWPOLYLINE
- **المحتوى**: جميع الأنابيب (قنوات الصرف) بما في ذلك عناصر الرسم الخاصة بالمقاطع الطولية

### 1.3 `EU 1_Regards_Habillage` — تسميات المنهولات (MTEXT)

- **النوع**: MTEXT (نص متعدد الأسطر)
- **المحتوى**: 97 تسمية منهول بأرقام R1-R97
- **التنسيق**: مفصول بـ `\P` (رمز فقرة MTEXT):

```
R1\PCT : 433.05\PCR : 430.75\PP : 2.30
```

**المعنى:**
- **R1**: رقم المنهول
- **PCT : 433.05**: Cote Terrain Naturel = 433.05 م
- **PCR : 430.75**: Cote Radier = 430.75 م  
- **PP : 2.30**: Profondeur = 2.30 م

**معلومة مهمة**: رُقُم المنهولات في التسميات (R1-R97) تختلف عن أرقام المقاطع الطولية (N104-N1, N55-N1, N135-N63). تسميات MTEXT تنتمي لشبكة مختلفة (على الأرجح شبكة `assai`).

**مقارنة الارتفاعات:**

| المصدر | الموقع | CT (أرض) | CR (قاع) |
|--------|--------|----------|----------|
| MTEXT R1 | (791809, 2171252) | 433.05 م | 430.75 م |
| مقطع N104→N1 | (791447, 2170807) | 406.16 م | 404.23 م |
| مقطع N55→N1 | (791778, 2171406) | 411.06 م | 408.65 م |

الفارق ≈ 27 مترًا — تأكيد أن التسميات لشبكة مختلفة.

### 2.3 ربط المقطع بالأنبوب الهندسي

**الخوارزمية:**
1. استخراج "Distances cumulées" من المقطع ← totalLength = آخر قيمة
2. البحث عن أنبوب (LWPOLYLINE) في `EU 1_Canalisations` بأقرب طول إجمالي
3. استخدام المسافات المتراكمة لتحديد مواقع العقد على الأنبوب (interpolation)
4. البحث عن أقرب INSERT (منهول) لكل موقع (ضمن 50 وحدة)
5. إنشاء مقاطع فردية بين العقد المتتالية

**نتائج المطابقة في Networks 1+2:**

| المقطع | طول المقطع | الأنبوب المطابق | فارق الطول | عدد العقد |
|--------|-----------|----------------|------------|-----------|
| N104→N1 | 1480.96 م | pipe[0] (1481 م) | 0.00 ✅ | 50 |
| N135→N63 | 917.59 م | pipe[3] (932 م) | 14.83 ⚠️ | 32 |
| N55→N1 | 1628.28 م | pipe[0] (1481 م) | 147.32 ❌ | 55 |

**مشكلة**: N55→N1 يطابق pipe[0] خطأ (لأن pipe[0] هو N104→N1). في Networks 1+2، مسار N55→N1 مقسم إلى 3 أنابيب: [4] (309م) + [3] (932م) + أجزاء أخرى. الحل هو مطابقة المسار الكامل (مجموع الأنابيب المتصلة) بدلاً من أنبوب واحد.

## 5. منهجية استخراج بيانات Covadis

### 5.1 خطوات الاستخراج الأساسية

1. **قراءة الملف**: استخدم TextDecoder('iso-8859-1') لأن الملف يحتوي على نصوص فرنسية مشكلة.

2. **تصفية الأنابيب الحقيقية**: استخدم معايير الإحداثيات ونسبة الأبعاد.

3. **استخراج المقاطع الطولية**: اقرأ نصوص `EU 1_PL_*_Textes` وابحث عن العناوين المعروفة.

4. **ربط المقاطع بالأنابيب**: طابق بطول المسافة المتراكمة أو بمواقع نقاط النهاية.

5. **استخراج المنهولات**: اقرأ جميع INSERTs من `EU 1_Regards`، وMTEXTs من `EU 1_Regards_Habillage`.

6. **تطابق المنهولات مع عقد المقاطع**: استخدم المسافة المتراكمة مع interpolation على الأنبوب المطابق.

7. **إنشاء مقاطع الأنابيب الفردية**: بين كل عقدتين متتاليتين في المقطع، احسب الطول (من Distances partielles) والانحدار (من فرق الارتفاع / الطول).

8. **تسميات الأنابيب**: اقرأ TEXTs من `EU 1_Canalisations_Habillage` وجمّعها. طابق التسميات الميدانية (وليست جدول الملخص) مع الأنابيب.

9. **الشبكات الأخرى**: اقرأ `DN 200` و `assai` و `New_EU 1_Canalisations_Pen_No__14`.

### 5.2 معالجة تنسيق MTEXT

نص MTEXT في Covadis:
```
R1\PCT : 433.05\PCR : 430.75\PP : 2.30
```
**الفاصل هو `\P`** (رمز فقرة MTEXT). يجب تقسيم النص على `\P`:

⚠️ **مهم في JavaScript**: يجب كتابة `'\\P'` (بخطين مائلين) لأن `'\P'` يعتبر escape غير معروف ويصبح `'P'` فقط.
```javascript
const parts = text.split('\\P');  // ✅ صحيح: \\ → \  → ينتج \P
// parts = ['R1', 'CT : 433.05', 'CR : 430.75', 'P : 2.30']

// const parts = text.split('\P');  // ❌ خطأ: \P يصبح P → split على حرف P فقط
// parts = ['R1\\', 'CT : 433.05\\', 'CR : 430.75\\', '', ' : 2.30']
```

ملاحظة: `\PP` ينتج `P : 2.30` (بدون PP) لأن الفاصل `\P` عند 28-29 يستهلك `\P` ويتبقى `P : 2.30`.

### 5.3 معالجة تنسيق TEXT لأنابيب

نص TEXT (مجموعتان متجاورتان ضمن 5 وحدات):
```
CR16-0315  30.00 ml    ← نص 1: المادة والقطر والطول
-0.84 %  <--           ← نص 2: الانحدار والاتجاه
```

```javascript
const diamRegex = /^(\S+)-(\d+)\s+([\d.]+)\s*ml?$/;  // CR16-0315  30.00 ml
const slopeRegex = /^([\-\d.]+)\s*%\s*(<--|-->)$/;     // -0.84 %  <--
```

## 6. مشاكل وحلول

### المشكلة 1: أنابيب المقطع الطولي مختلطة مع الحقيقية
- **الحل**: استخدم 3 معايير للتصفية (الإحداثيات + الأبعاد + نسبة الطول).
- **نتيجة**: 13 من 36 أنبوبًا تبقى (36% فقط).

### المشكلة 2: تسميات جدول الملخص (96 مجموعة) مختلطة مع التسميات الميدانية
- **الحل**: المجموعات 0-16 فقط (عند x≈791806) هي تسميات ميدانية. البقية هي جدول ملخص.
- **التعرف على التسميات الميدانية**: قرب أنبوب حقيقي (ضمن 50-100 وحدة)، خارج منطقة جدول الملخص (x < 791900).

### المشكلة 3: تسميات المنهولات (R1-R97) لا تطابق INSERTs
- **السبب**: تسميات R1-R97 تنتمي لشبكة `assai` وليس لـ EU 1_Regards.
- **الحل**: ابحث عن INSERTs مطابقة في `assai` layer.

### المشكلة 4: N55→N1 يطابق أنبوبًا خاطئًا
- **السبب**: في Networks 1+2، مسار N55→N1 مقسم عبر 3 أنابيب.
- **الحل**: استخدم مطابقة المسار الكامل (مجموع الأنابيب المتصلة).

### المشكلة 5: ترميز ISO-8859-1
- يجب استخدام `TextDecoder('iso-8859-1')` بدلاً من UTF-8.
